## v3.0 핵심 규칙

1. 시민과 마피아 모두 개인전이며, 마지막 생존자 1명이 승리한다.
2. 플레이어는 등에 유효 메모가 3장 붙는 순간 즉시 사망한다.
3. 투표, 추방, 시민 팀 승리, 최종재판은 현행 규칙에서 제거한다.
4. 회의는 처형 단계가 아니라 정보 공유와 다음 표적에 대한 여론 형성 단계다.
5. 마피아는 색상 위조와 빠른 메모 획득을 사용한다.
6. 마피아 유전은 선택 규칙이며, 권장 기본값은 1회·15초·생존자 3명 이하 비활성이다.
7. 아이템 프레임워크는 Alpha, 카메라·로프·힐·투명화 주스는 Beta 검증 대상으로 관리한다.
8. 레벨업·전생은 Season 1 범위이며, 공식 경쟁 환경에서는 능력치 보정을 비활성화하는 방향을 우선한다.

## 운영 방식

1. 각 백로그의 담당자, 검토자, 스프린트, 기한을 계획 회의에서 확정한다.
2. 외부 이슈 추적 도구로 이전할 때 통합 파일의 `11_Jira_가져오기`를 사용한다.
3. `완료`는 구현, 코드 리뷰, 정상·예외 QA, 증빙 연결, 관련 사양 갱신이 모두 끝난 상태다.
4. 승패, 마피아 유전, 비공개 정보, 서버 프로토콜, 보이스 저장, 성장 능력, 확률형 상품은 의사결정 기록 없이 변경하지 않는다.
5. 자기 등 메모의 내용·색·개수와 실제 작성자·위조 여부는 공개 클라이언트 데이터에 포함하지 않는다.
6. 구버전의 `night_vote`, `final_trial`, `exile` 관련 설정과 테스트는 폐기하거나 v3.0 회귀 방지 케이스로 전환한다.

## 상태 운영

`미착수 → 사양 확인 중 → 구현 중 → 코드 리뷰 → QA 진행 중 → 완료`

외부 의존으로 진행할 수 없으면 `차단`, 범위에서 제외하면 `보류`로 기록한다.

## 주요 미확정 의사결정

- 동시 사망으로 생존자가 0명이 된 경우의 결과
- 마피아 유전 기본값과 반복 허용 여부
- 힐의 메모 제거 순서
- 사망 시 역할 공개 범위
- 자유 입력 메모의 출시 범위
- 보이스 증거 저장 여부와 지역별 동의 정책
- 레벨업 능력치의 공식 매치 적용 여부
- 유료 확률형 상품 도입 여부

## 수치 사용 주의

평균 경기 시간, 역할별 승률, 위험자 생존률, 아이템 승률 편차, 크래시 프리 목표는 초기 기준이다. Alpha·Beta 실측 후 RC 승인값으로 갱신한다.
